# pi-dask 接口契约

## 定位

`dask` 是 Pi 的结构化提问工具，只收集平铺、互不依赖的可枚举选择。调用方负责问题逻辑、顺序、推荐、条件分支与收束。

当调用方确认确实需要用户决策，并已将问题收敛为单选 2–5 项或复选 2–12 项时，应调用 `dask`，而不是在回复正文中直接列出编号选项；即使当前只有一道题也一样。开放式探索以及调用方本应自行收口的工程细节不进入 `dask`。

## 请求

```ts
type Scalar = string | number | boolean;

type Question = {
  id: string;
  title: string;
  description: string;
  type: "single" | "multiple";
  labels: Array<{
    id: number;
    label: string;
    value: Scalar;
    description: string;
  }>;
};

type DaskRequest = {
  questions: Question[];
};
```

约束：

- 调用方应将调用时已知且互不依赖的问题合并进同一个 `questions` 数组；若某个答案会改变另一题是否出现、题意或选项，则必须在获得该答案后另行调用。答案按输入顺序返回。
- `id` 是调用方提供的稳定问题标识符；同一次请求内必须唯一。
- 选项 `id` 是正安全整数，且必须与 `labels` 中的位置一致：第一项为 `1`、第二项为 `2`，依此类推。同题中不得缺号、跳号或重排。
- `type: "single"` 的 `labels` 数量为 2–5；`type: "multiple"` 为 2–12。
- `labels[].id` 是 TUI（终端界面）中展示在选项前的序号；调用方和用户可在“其他（自行填写）”中用该序号指代选项。同题 `labels[].value` 必须唯一。
- `value` 仅允许字符串、数字或布尔值；不接受对象或数组。多模态资源由调用方以文本路径或 URL（统一资源定位符）表达。
- 每道题固定提供“其他（自行填写）”入口，不把它放进 `labels`。
- 不支持独立自由文本题、条件跳题、长选项搜索或持久化状态。

## 返回

确认提交时返回：

```ts
type AnswerValue =
  | { source: "label"; value: Scalar }
  | { source: "custom"; value: string };

type Answer = {
  id: string;
  value: AnswerValue[];
};

type DaskResult = {
  answers: Answer[];
};
```

- `answers` 与 `questions` 保持相同顺序，每个问题恰有一个答案对象。
- 单选的 `value` 恰有一项；复选至少一项，最多为选项数加一（最多一项 `source: "custom"`）。
- `source: "label"` 的 `value` 原样来自所选 `question.labels[].value`，不返回选项 ID，也不派生 `value_id`。
- `source: "custom"` 的 `value` 是用户在“其他”入口填写的非空文本。

## 向导行为

- 逐题展示题目内容、题目说明、选项序号、选项内容和选项说明；选项序号直接使用 `labels[].id`。
- 单题交互只展示题目与选项，不显示无意义的多题导航。
- 多题交互显示“当前题 / 总题数”和全部题号：当前题使用高亮背景，已回答题显示 `✓`，未回答题显示 `○`。
- 在问题页面，数字键 `1`～`9` 直跳到存在的对应题，`←` / `→` 在相邻题之间移动且不越过首尾；第 10 题及以后仍可通过左右键访问。数字键只用于切题，不是选项选择快捷键；编辑“其他”文本时仍作为普通文本输入。
- 允许进入尚未回答的题目；切题会保留各题已选答案和自填文本。`↑` / `↓` 始终只移动当前题的选项焦点。
- 单选用上下移动和回车选择；复选用上下移动、空格切换，回车继续。按原逐题路径完成一题后仍进入下一题。
- 最后一页展示按题目输入顺序排列的答案摘要。存在未回答题时，确认操作返回第一道未回答题补答，不生成 `DaskResult`；全部回答后确认才提交结果。
- 在任意阶段取消都以工具取消/错误结束，不返回部分答案，不持久化中途状态。
- 非 TUI（终端界面）模式不执行交互，工具以错误结束。
