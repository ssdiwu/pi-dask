# AGENTS.md — pi-dask

## 一句话

pi-dask 是 Pi 的结构化提问扩展：收集可枚举的单选/复选答案；不决定问题逻辑，不实现条件问卷。

## 先读

1. `README.md` — 产品定位、已确认边界与尚未做项
2. `doc/README.md` — 文档地图
3. `doc/术语表.md` — 项目术语
4. 实现出现后，阅读目标代码目录的 `README.md`

## 代码边界

- 公开 npm（Node 包注册表）包名为 `pi-dask`，模型工具名为 `dask`。
- 单选仅允许 2–5 项；复选仅允许 2–12 项。超过上限必须由调用方先收敛，不用 UI（用户界面）补救。
- 每题提供“其他（自行填写）”；不提供独立自由文本题。
- 一次调用可收集多道平铺、互不依赖的问题；不做条件跳题、问卷状态或持久化。
- dask 只负责结构化收集。`507-grill`（决策树追问）等调用方负责问题顺序、候选质量、推荐与收束；不得把这些策略写进 dask。
- 不硬编码密钥、Token（令牌）或本机绝对路径。

## 验证

实现前先为交互与结果结构建立确定性测试。实现后按项目已有分层命令执行；没有测试命令时先补最小可重复反馈环。

## 工程纪律

- 只有第三次真实重复才抽象；不为想象中的问卷能力提前建接口。
- 函数超过 100 行时考虑拆分。
- 测试验证公开行为，mock（模拟）只放系统边界。
- 临时调试日志必须使用唯一标记，完成前清理。
- 一次 Git commit（Git 提交）只做一件事，使用 `Conventional Commits`（约定式提交）格式。
